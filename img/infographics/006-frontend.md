## Main Content

Create a polished PNG infographic in English that explains the Krateo Frontend as a standalone experience layer for stakeholders. This version should present the frontend primarily as the user-facing runtime UI engine, not as an isolated backend service. The audience is non-technical and semi-technical stakeholders.

Use case: infographic-diagram
Asset type: stakeholder-facing frontend standalone explainer
Aspect ratio: landscape 16:9
Style: clean enterprise platform infographic, very light background, structured card layout, premium product UX aesthetic, polished vector-like illustrations, blue/teal/orange accents, excellent typography.

Main title: "What Krateo Frontend Does"
Subtitle: "The experience layer for dynamic, authenticated platform interactions"

Message framing requirements:
- Treat the frontend as the user-facing runtime layer.
- Emphasize what users see and what the frontend orchestrates visually.
- Mention external services only as supporting capabilities, not as the main story.
- Make the frontend feel like the central experience hub.

Layout requirements:
- Top summary band with 3 short statements:
  - "Presents the platform experience"
  - "Loads dynamic UI at runtime"
  - "Guides users through actions and live feedback"

- Central large card or flow focused on the frontend lifecycle with these stages:
  1. "Load runtime config"
  2. "Authenticate user"
  3. "Load routes and widgets"
  4. "Render pages and forms"
  5. "Execute user actions"
  6. "Reflect live updates"

- Left card titled "What users get" with bullets:
  - "Login and profile experience"
  - "Navigation and page structure"
  - "Forms, tables, charts, and widgets"
  - "Notifications and live operational feedback"

- Right card titled "What the frontend orchestrates" with bullets:
  - "Runtime configuration"
  - "Session and access state"
  - "Dynamic widget rendering"
  - "Action execution and event refresh"

- Small supporting strip or secondary panel titled "Powered by platform services" with 3 concise items:
  - "AuthN for authentication"
  - "Snowplow for routes, widgets, and actions"
  - "Events Presenter for live updates"

- Panel titled "Why it matters" with 4 concise points:
  - "One UI can support many platform workflows"
  - "Behavior can evolve without rewriting every screen"
  - "Platform definitions can directly shape the experience"
  - "Users get dynamic and responsive operational journeys"

- Panel titled "Watch-outs" with 4 concise points:
  - "Poor runtime configuration degrades UX"
  - "Broken widget definitions surface at runtime"
  - "Latency affects perceived responsiveness"
  - "Service dependencies influence availability"

- Footer strip text:
  "User journey: configure -> sign in -> navigate -> act -> receive feedback"

Visual requirements:
- Keep the frontend visually dominant in the composition
- Use supportive smaller icons for auth, widgets, actions, events, config
- Do not make the backend service topology the main focus
- Keep text exact, concise, and fully legible
- No logos, no screenshots, no watermarks, no code blocks
